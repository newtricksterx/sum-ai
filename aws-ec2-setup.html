AWS EC2 Setup — Sum AI

Instance: t3.micro (2 vCPU, 1 GB RAM) • OS: Ubuntu 24.04 LTS

Step 1 — Launch Instance

  1. Go to EC2 → Launch Instance in the AWS Console
  2. Name: sum-ai
  3. AMI: Ubuntu 24.04 LTS (free tier eligible)
  4. Instance type: t3.micro
  5. Key pair: Create new → download the .pem file
  6. Network settings → Create security group:
PortSourcePurpose
22 (SSH)My IPSSH access
80 (HTTP)0.0.0.0/0HTTP → HTTPS redirect
443 (HTTPS)0.0.0.0/0API traffic
  1. Storage: 20 GB gp3
  2. Click Launch instance

Step 2 — Elastic IP

Prevents your public IP from changing on instance stop/start.

  1. EC2 → Elastic IPs → Allocate
  2. Select the new IP → Actions → Associate → pick your instance
  3. Note this IP — this is your permanent server address

Step 3 — Point Your Domain

In your domain registrar (Namecheap, Cloudflare, etc.):

  • A record → @ → <Elastic IP>
  • A record → www → <Elastic IP> (optional)

Wait a few minutes for DNS propagation.

Step 4 — SSH In

chmod 400 your-key.pem
ssh -i your-key.pem ubuntu@<Elastic IP>

Step 5 — Install Docker

# Update system
sudo apt update && sudo apt upgrade -y

# Install Docker
sudo apt install -y docker.io docker-compose-v2

# Let your user run docker without sudo
sudo usermod -aG docker $USER

# Log out and back in
exit

SSH back in and verify:

ssh -i your-key.pem ubuntu@<Elastic IP>
docker --version

Step 6 — Clone Your Repo

git clone https://github.com/YOUR_USER/sum-ai.git
cd sum-ai

Step 7 — Create Production Env File

nano backend/.env.prod

Paste the following (replace placeholder values):

PRODUCTION=True
DJANGO_SECRET_KEY=<run: python3 -c "import secrets; print(secrets.token_urlsafe(50))">
GEMINI_API_KEY=<your-gemini-key>
CORS_ALLOWED_ORIGINS=https://yourdomain.com

POSTGRES_DB=sumai
POSTGRES_USER=sumai_user
POSTGRES_PASSWORD=<run: python3 -c "import secrets; print(secrets.token_urlsafe(32))">

Add any other env vars your settings.py expects (Stripe keys, etc.).

Step 8 — SSL Certificates (Let's Encrypt)

# Install certbot
sudo apt install -y certbot

# Get cert (nothing should be running on port 80 yet)
sudo certbot certonly --standalone -d yourdomain.com

# Copy certs into your project
sudo cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem nginx/certs/
sudo cp /etc/letsencrypt/live/yourdomain.com/privkey.pem nginx/certs/
sudo chown $USER:$USER nginx/certs/*.pem

Step 9 — Build and Launch

cd ~/sum-ai
docker compose -f docker-compose.prod.yml up -d --build

Watch the logs until everything is healthy:

docker compose -f docker-compose.prod.yml logs -f

Press Ctrl+C to exit logs.

Step 10 — Verify

# From the server
curl -k https://localhost/api/auth/csrf

# From your browser
# https://yourdomain.com/api/auth/csrf

Step 11 — Auto-Renew SSL

crontab -e

Add this line:

0 3 1,15 * * certbot renew --quiet && cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem ~/sum-ai/nginx/certs/ && cp /etc/letsencrypt/live/yourdomain.com/privkey.pem ~/sum-ai/nginx/certs/ && docker compose -f ~/sum-ai/docker-compose.prod.yml restart nginx

Renews certs and restarts Nginx on the 1st and 15th of each month.

Step 12 — Enable Swap (safety net for 1 GB RAM)

sudo fallocate -l 1G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

Prevents OOM-kills if memory spikes temporarily.
